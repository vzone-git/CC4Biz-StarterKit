# Example Prompts

Copy-paste these into Claude Code to get started. Replace the bracketed parts with your details.

---

## Research

### Deep Research Report
```
Research [topic]. I need a report with:
- Executive summary
- Key findings with sources
- Implications for [my company/situation]
- Risks and counterarguments
- Actionable recommendations
```

### Competitive Analysis
```
Do a competitive analysis of [Company A] vs [Company B] vs [Company C].
Compare: pricing, features, target market, funding, traction, and key differentiators.
Focus on what matters for [my specific situation/decision].
```

### Market Sizing
```
Help me size the market for [product/service].
Build TAM/SAM/SOM (Total Addressable Market / Serviceable Addressable / Serviceable Obtainable) estimates with explicit assumptions.
Include both top-down (industry reports) and bottom-up (customer math) approaches.
```

### ROI Analysis
```
Build an ROI analysis for [investment/partnership/tool].
Costs: [list known costs]
Expected benefits: [list expected benefits]
Timeline: [timeframe]
Include a conservative, base, and optimistic scenario.
```

## Presentations

### Pitch Deck
```
Create a pitch deck for [company/product].
Here's our key info:
- What we do: [one sentence]
- Problem: [what pain we solve]
- Traction: [key metrics]
- Ask: [what we're raising / what we want]

[Paste any additional notes, bullet points, or rough content]
```

### Business Review / Board Deck
```
Build a quarterly business review presentation covering:
- Key metrics: [paste numbers]
- Wins this quarter: [list]
- Challenges: [list]
- Plan for next quarter: [list]
- Decisions needed from the board: [list]
```

### Partnership Proposal
```
Create a presentation proposing a partnership between [our company] and [partner].
Value proposition for them: [why they should care]
Value proposition for us: [what we get]
Key terms: [deal structure]
Save it as output/presentations/[partner]-proposal.html
```

## Investor Materials

### Investor Memo
```
Write an investor memo for our [seed/Series A/etc.] raise.
Company: [name and one-line description]
Traction: [key metrics]
Team: [founders and relevant background]
Market: [who we sell to and how big it is]
Raise: [amount, instrument, use of funds]
```

### Cold Outreach to Investor
```
Draft a cold email to [investor name] at [fund name].
Why they're relevant: [their thesis, portfolio companies, or recent investments that connect to us]
Our one-liner: [what we do]
Our proof point: [strongest traction metric]
The ask: [meeting, intro, feedback]
```

### Financial Model
```
Build a financial model for [company/product].
Current revenue: [number]
Growth assumptions: [what you expect]
Key cost drivers: [list]
Include: monthly projections for 18 months, annual for 3 years
Show: bear/base/bull cases
```

## Writing & Content

### Blog Post / Article
```
Write a blog post about [topic].
Audience: [who reads this]
Goal: [what should the reader think/do after reading]
Tone: [professional, conversational, technical, etc.]
Length: [short ~500 words / medium ~1000 / long ~2000]
Key points to cover: [list]
```

### LinkedIn Post
```
Write a LinkedIn post announcing [news/insight/milestone].
Key message: [the one thing people should take away]
Tone: [professional but human, data-driven, etc.]
Include a call to action: [what should people do]
```

### Newsletter Issue
```
Write a newsletter issue about [topic/theme].
Sections: [list 2-4 sections]
Audience: [who subscribes]
Past issues have been about: [brief context]
```

## Business Cases & Strategy

### Business Case Document
```
Write a business case for [proposed initiative].
Problem statement: [what's broken or missing]
Proposed solution: [what we want to do]
Expected outcomes: [what success looks like]
Costs: [what it will take]
Risks: [what could go wrong]
Timeline: [when we'd see results]
```

### Partnership Evaluation
```
Evaluate whether we should partner with [company].
Research them: funding, traction, reputation, strategic fit.
Analyze: what we'd gain, what we'd give up, deal structure options.
Recommend: go/no-go with reasoning.
```

### Vendor Comparison
```
Compare [Vendor A] vs [Vendor B] vs [Vendor C] for [our use case].
Evaluation criteria: [list what matters - price, features, support, security, etc.]
Our specific needs: [describe]
Deliver as a scored comparison matrix with a recommendation.
```
